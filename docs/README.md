# Docs Directory

这里放的是“给人看的长期文档”，不是运行时代码。

## 这类文档应该解决什么问题

- 项目结构为什么这样设计
- 某项能力应该落到哪一层
- 开发时有哪些稳定约定
- 新成员第一次进入项目时应该先读什么

## 当前文档索引

### [development.md](development.md)

项目的开发组织规范，重点说明分层边界、放置规则、测试约定和变更原则。

### [../environment.yml](../environment.yml)

用于创建统一的 conda 开发环境 `classbot-mcp`。

## 编写建议

这个目录下的文档不要求统一格式，但建议遵守下面几条：

1. 优先写“为什么这样做”，其次才是“怎么做”。
2. 如果文档只对某个子目录有效，优先把说明写到那个子目录自己的 README，而不是堆在这里。
3. 如果文档会随着实现频繁变化，尽量引用代码路径，减少复制粘贴式描述。

## 什么时候新增这里的文档

- 结构规则开始变复杂
- 新增跨目录的通用约定
- 需要为协作者提供项目级决策背景
