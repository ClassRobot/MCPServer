# Source Layout

`src/` 是源码根目录，它的存在不是为了多包一层，而是为了把“源码”和“仓库其他内容”明确分开。

## 为什么有这一层

- 避免根目录下的脚本、测试、配置文件和可导入模块混在一起
- 让本地开发更接近真实安装后的导入行为
- 让后续打包、editable install、测试路径控制更稳定

## 这里应该放什么

- 可被 Python 导入的项目源码
- 业务模块、配置模块、装配模块
- 与源码强绑定的包级 README

## 这里不应该放什么

- 项目说明文档
- 一次性调试脚本
- 测试文件
- 构建输出和运行缓存
- 项目内虚拟环境目录

## 当前包

- [`mcp_server/`](mcp_server/README.md): 当前项目的主包，承载 MCP Server 的配置、装配和能力注册。
