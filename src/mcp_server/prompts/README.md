# Prompt Authoring Notes

这个目录放 MCP `prompt`，重点不是“能返回一段字符串”，而是“给客户端或模型提供稳定、可复用的提示模板”。

## 写 prompt 时，建议先回答这三个问题

- 它服务的任务场景是什么
- 它需要哪些最少输入
- 它输出时希望把模型引导到什么结构

## 当前目录适合的写法

这里更适合“模板作者视角”的说明，而不是接口文档口吻。

一个好的 prompt 模块通常会：

- 把输入参数命名得足够语义化
- 明确输出目标，而不是堆砌约束
- 避免把项目瞬时细节写死进模板

## 维护提醒

- 如果 prompt 开始依赖大量业务上下文，说明可能需要先整理 resource 或 service，再回来写 prompt。
- 如果多个 prompt 共享同一种输出结构，可以提炼共同约束，但不要为了复用牺牲可读性。
