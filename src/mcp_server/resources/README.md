# Resources Reference

这个目录管理 MCP `resource`，也就是供客户端读取的稳定信息源。

## 资源更像什么

它更像“可读地址”而不是“被执行的命令”。

例如：

- `project://info`
- `docs://...`
- `config://...`

## 编写资源时要保持的特性

- 地址命名稳定
- 内容语义清楚
- 输出适合阅读或被模型消费
- 不把副作用逻辑塞进资源读取

## URI 命名建议

- 用统一前缀表达资源域
- 避免同一类信息出现多个接近但含义不同的 URI
- 如果资源会长期被客户端配置引用，后续不要随意改名

## 适合放进这里的内容

- 项目说明
- 静态元信息
- 某类数据的只读视图

## 不适合放进这里的内容

- 明显带副作用的操作
- 需要复杂流程编排的逻辑
- 其实应该做成 tool 的能力
