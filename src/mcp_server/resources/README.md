# Resources Guide

本目录管理所有供客户端读取的只读静态或状态信息源（Resources）。

## 核心设计理念
- **无副作用的只读地址**: Resource 应设计为“只读数据视图”而非“可执行命令”（例如 `project://info`），严禁触发有外部写副作用的操作。
- **稳定 URI 命名**: 使用统一的前缀定义独立的资源空间（Schema），保证 URI 在客户端配置文件中长期可用。

## 适用范围划分
- **适用场景**: 静态信息汇总、运行时只读元数据、配置文件的快照视图。
- **禁用场景**: 包含写副作用的操作、需要复杂业务流转的动作。此类型能力应优先考虑包装为 Tools。
